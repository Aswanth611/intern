Database Schema Overview

  • tracks $\rightarrow$ modules $\rightarrow$ micro_lessons: Holds the course structure. Static catalogs referenceable by active student courses.
  • students $\rightarrow$ progress: Stores the state of each student's current position (e.g. Locked, Active, Completed).
  • progress $\rightarrow$ micro_lessons: Resolves which specific lessons are marked completed or unlocked.
  • students $\rightarrow$ quiz_results $\leftarrow$ modules: Logs records of score achievements to gate unlocking of subsequent modules.

Automation Logic Specifications

This technical reference guide outlines the specific backend rule engine that handles locking, progress, and redirection.

Event Trigger Pre-Conditions Automated System Actions
Track Selected
  • Student finishes onboarding form.
  • Valid student name & email.
  • Insert record in students table.
  • Fetch course structure from modules and micro_lessons.
  • Populate progress tracker rows with "locked" status.
  • Set Module 1, Lesson 1 status to "active" (unlocked).
  • Redirect student from signup form to learning roadmap dashboard.
Lesson Completed
  • Student clicks "Complete Lesson" on an active lesson.
  • Current lesson status is "active".
  • Update current lesson progress status to "completed".
  • Update completed_at timestamp.
  • Verify if next lesson exists in the current module.
  • If yes: set next lesson status to "active" (unlocked).
  • If no: check if all 5 module lessons are completed.
Module Completed
  • All 5 lessons of active module have "completed" status in database.
  • Trigger redirect to module assessment quiz interface.
  • Restrict continuing to further lessons until quiz passed.
Quiz Passed
  • Student scores greater than 60% (2/3 correct) on module quiz.
  • Subsequent module exists in selected track.
  • Insert record to quiz_results table (passed = true).
  • Unlock subsequent module.
  • Unlock lesson 1 of subsequent module, setting status to "active".
  • Refresh student dashboard with next target action.
Quiz Failed
  • Student scores $< 60\%$ on module quiz.
  • Insert record to quiz_results table (passed = false).
  • Keep next module locked.
  • Display retry options and review links.
